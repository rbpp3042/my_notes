Циклы агентов на практике

Самый удачный пример применения агентских циклов у меня на данный момент — это работа с кодом.

В целом это самая удобная и благодатная среда, поскольку сама разработка уже подразумевает цикличность. Классические процессы (ревью, тесты, проверки) — это и есть циклы. Вопрос только в том, кто их запускает.

Связка

Я настроил связку между Claude Code и агентом Hermes на open-weight модели (сейчас экспериментирую с GLM — китайская модель с открытыми весами, интересно следить за их развитием).

Получилось два агента с разделением ролей:

Claude CodeHermes + GLM
РольПланировщик и ревьюерИсполнитель
Что делаетПишет план разработки, делает code reviewПишет код по готовому плану

Цикл

  1. Claude Code пишет детальный план изменений и кладёт в репозиторий
  2. Hermes (на GLM) пишет и правит код. По подробному плану от Claude это несложно
  3. Hermes сам открывает Pull Request
  4. PR автоматически запускает code review — его выполняет Claude Code. Он либо одобряет, либо присылает комментарии
  5. Отчёт приходит прямо в открытый PR, а также в мой чат с Hermes в Telegram — поэтому я тоже остаюсь в курсе
  6. Оттуда я могу попросить Hermes выполнить правки и добавить новый коммит в тот же PR
  7. Если Claude Code даёт ОК — я позже вручную смерджу PR и сделаю деплой

Этот цикл можно сделать ещё более независимым от меня, но пока я остановился на этой версии.

Почему open-weight модель справляется

GLM не пишет код с нуля и не придумывает архитектуру. Она работает по плану, который уже составил Claude Code. Это сильно снижает требования к модели — тут ей нужен не креатив, а аккуратность.

Где это работает

Пока внедрил в одном проекте, который хорошо знаю и по которому накоплено много контекста. Там настроен харнесс и понятно, как его дорабатывать.

Постепенно планирую затягивать эту схему в другие проекты.

При хорошей подготовке можно управлять разработкой через Telegram-чат, открывая терминал только при необходимости.
